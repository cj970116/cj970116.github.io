深入理解js中的逻辑运算

为什么突然回想到要研究js的逻辑运算呢,你可能会说逻辑运算不很简单吗,真||假为真,真&&真为真…,没错我们经常用这些来判断if条件, 但是,这些逻辑运算只能用在条件判断里面吗…在写这篇博客之前,我也是这样想的,直到我看到了这个代码

function Difference(arr = [], oarr = []) {
    return arr.reduce((t, v) => (!oarr.includes(v) && t.push(v), t), []);
}
const arr1 = [1, 2, 3, 4, 5];
const arr2 = [2, 3, 6]
Difference(arr1, arr2); // [1, 4, 5]

这个reduce我将在后续的博客中继续探究。第一次看到&&用在这个位置让我有点好奇，于是去百度查阅了资料,发现

a || b：如果a是true，那么b不管是true还是false，都返回true。因此不用判断b了，这个时候刚好判断到a，因此返回a。如果a是false，那么就要判断b，如果b是true，那么返回true，如果b是false，返回false，其实不就是返回b了吗。 a && b：如果a是false，那么b不管是true还是false，都返回false，因此不用判断b了，这个时候刚好判断到a，因此返回a。如果a是true，那么就要在判断b，和刚刚一样，不管b是true是false，都返回b。

然后，我做了几个例子

let a= 'char'
let b=0
let c= new Object()
let ret0 = a&&b

console.log(ret0); //0

所以，我们得出结论：a&& b :如果执行a后返回true，则执行b并返回b的值；如果执行a后返回false，则整个表达式返回a的值，b不执行； a || b :如果执行a后返回true，则整个表达式返回a的值，b不执行；如果执行a后返回false，则执行b并返回b的值； && 优先级高于 ||;

function Difference(arr = [], oarr = []) {
    return arr.reduce((t, v) => (!oarr.includes(v) && t.push(v), t), []);
}
const arr1 = [1, 2, 3, 4, 5];
const arr2 = [2, 3, 6]
Difference(arr1, arr2); // [1, 4, 5]

Js reduce()函数:

形式：array.reduce((t, v, i, a) => {}, initValue) 参数:

  • callback：回调函数(必选)
  • initValue：初始值(可选)

回调函数的参数:

  • total(t)：累计器完成计算的返回值(必选)
  • value(v)：当前元素(必选)
  • index(i)：当前元素的索引(可选)
  • array(a)：当前元素所属的数组对象(可选)

Js逗号表达式:

逗号运算符的作用是将若干表达式连接起来。它的优先级别在所有运算符中是最低的，结合方向是"自左至右"的。 （即：逗号运算符：是按顺序执行表达式，并且获得右边表达式的值。

上面代码中，oarr和arr的初始值都为空数组,传入参数arr1和arr2，则第一次!oarr.includes(v)为true，则这个表达式必然返回t.push(v)的结果,然后由于逗号表达式，返回t。那么只要arr1中的元素在arr2中找不到，则把这个元素push到数组t中，并返回这个数组，这样就实现了数组的过滤
